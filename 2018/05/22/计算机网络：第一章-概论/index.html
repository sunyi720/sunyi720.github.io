计算机网络：第一章-概论

发表于 2018-05-22 | 分类于 计算机网络 | 阅读次数:

1 因特网的组成

  • 边缘部分:由英特网中的主机组成，这些主机又称为端系统。
  • 核心部分:由大量的网络和路由器组成，为边缘部分提供服务。

1.1 边缘部分

边缘部分利用核心部分提供的服务,进行进程间的通信。

1.1.1 C/S方式

一台主机作为客户端，发送请求。一台主机作为服务端，接受请求。服务端为客户端提供服务。

客服端需要知道服务端进程地址，不需要强大的硬件和操作系统。
服务端启动后持续不断的工作，需要强大的硬件和操作系统。

1.1.2 P2P方式

每一台主机都可以发送和接受请求。

1.2 核心部分

三种交换方式：

  • 电路交换：一开始就建立好连接，并且整个过程中都占用线路资源。
    • 优：对于大型数据性能好。
    • 劣：传输效率低。
  • 分组交换：将数据封装成分组，采用存储转发的方式经路由器传输。
    • 优：信道的传输效率效率高。
    • 劣：时延增加，带宽不定。
  • 报文交换：同样采用了存储转发的方式，只不过没有分解成分组。

2 计算机网络的性能

2.1 计算机网络的性能指标

  1. 速率：数字信道上传送数据的速率，也称为数据率或比特率，单位b/s。
  2. 带宽：（1）原指信号中的频率宽度。（2）计算机中指单位时间内从网络中的某一点到另外一点的最大数据率，单位b/s。
  3. 吞吐量：单位时间内通过某个网络的数据量。

  4. 时延：数据从网络中的一端传送到另一端的时间，由一下几个部分组成：

    1. 发送时延：主机或者路由器发送数据所需要的时间。发送时延=数据帧长度（b）/信道带宽(b/s)。
    2. 传播时延：电磁波在信道中传播一定的距离所需要的时间。传播时延=信道长度(m)/电磁波在信道上的传播速率(m/s)。

    对于高速网络链路，我们提高的仅仅是数据的发送速率，而不是比特在链路上的传播速率。通常所说的光纤的传输速率高，是指发送速率高，而传播速率比铜线的传播速度还要略低。

    1. 处理时延：主机或者路由器收到分组时的处理时间。
    2. 排队时延：分组进出路由器都要排队等待处理。
  5. 时延带宽积：以比特为长度的链路长度。时延带宽积=传播时延*带宽。
  6. 往返时间：数据从发出，到接受到确认的时间。
  7. 利用率：有信道利用率和网络利用率：
    1. 信道利用率：信道百分之几的时间有数据通过。
    2. 网络利用率：全网的信道利用率的加权平均值。

D0表示网络空闲时的时延(即U=0，该值不为空)，D表示当前的时延，U表示网络利用率。则有D=D0/(1-U).

2.2 非性能特征

费用，质量，标准化，可靠性，可拓展性和可升级性，易于管理和维护。

3 计算机网络的体系结构

3.1 计算机网络体系结构的形成

实现两台计算机之间的数据传输，至少需要以下几个工作：

  1. 激活数据通信的通路。
  2. 识别接受数据的计算机。
  3. 检查对方计算机是否开机
  4. 确认对方是否已经做好接受数据和保存数据的准备工作。
  5. 若数据不兼容，至少一个计算机需要做格式转换的工作。
  6. 确保数据的正确传输。

完成计算机的数据传输，需要两台计算机高度协调，这种协调比较“复杂”，因此出现了分层的方法。
根据分层的方法产生了系统网络体系结构（SNA），但是最初每个公司都有自己的一套SNA，为了防止垄断，国际标准化组织ISO诞生，并制定出了著名的开放系统互连参考模型（SOI/RM） ，但是ISO并没有在因特网上并没有实行，原因如下：

  1. 缺乏经验，缺乏商业驱动力。
  2. 复杂，效率低。
  3. 制定周期太长。
  4. 层次划分不合理

得到最广泛运用的不是法律上的国际标准OSI/RM,而是非国际标准TCP/IP。

3.2 协议与划分层次

为进行网络中的数据交换而建立的规则，标准或约定成为网络协议。由以下三个要素组成：

  1. 语法：数据与控制信息的结构或格式。
  2. 语义：需要发出的控制信息，动作以及响应。
  3. 同步：事件实现顺序的详细说明。

举个例子说明分层的意义，假定两台电脑需要数据传输，首先需要（1）确认对方计算机做好了接受数据和存储数据的准备。（2）其中一台计算机完成数据格式转换的功能。这两个功能我们用文件传输模块来实现，之后文件和文件传输命令我们用低一层的通信服务模块来实现，同样我们定义一个再低一层的网络接入模块，负责网络接口细节处理，并为上级提供服务。

分层的优：

  1. 独立：各层间不影响，仅需要使用提供的接口。
  2. 灵活性好：当一层发生变化，不会对其他层产生影响。
  3. 结构上可以分割开：各层都可以采用最合适的技术实现。
  4. 易于实现和维护。
  5. 促进标准化工作。

分层的劣：有些功能多层重复，产生了额外开销。

3.3 具有五层协议的体系结构

  • 应用层：为进程提供服务。HTTP协议，SMTP协议，FTP协议。
  • 运输层：为进程之间的通信提供服务。运输层主要使用以下两种服务：
    1. 传输控制协议TCP：面向连接的，数据传输的单位是报文段，能够提供可靠的交付。
    2. 用户数据报协议UDP：无连接的，数据传输的单位是用户数据报，不提供可靠的交付，只提供“尽最大努力”的交付。

  • 网络层：负责为分组交换网上的不同主机提供服务。在发送数据时，网络层把运输层产生的报文段或者是用户数据报封装成分组或包进行传输。在TCP/IP中，由于网络层使用的是IP协议，因此也叫IP数据报，本文中把“分组”和“包”作为同义词。网络层另一个主要功能是为运输层传下来的分组通过路由器找到目的主机。

    需要注意，无论在哪一层的数据单元，都可以用“分组”来笼统的概括。

  • 数据链路层：两个主机之间的数据传输，是在一段一段的链路上传输的，也就是说，在相邻节点之间（主机和路由器，路由器和路由器），数据是直接传送的。数据链路层将网络层的IP数据报组装成帧，在相邻节点之间“透明”传输，每一帧包含了数据和控制信息（同步信息，地址信息，控制信息）。控制信息可以检测信息有无差错，若数据有差错，数据链路层就将数据丢弃，若需要修改，则由运输层的TCP协议来完成。

    所谓透明，是指无论什么比特组合的数据都可以通过数据链路层。

  • 物理层：透明的传输比特流。如用多大的电压表示0，1；接收方如何做分辨；

2.4 实体,协议,服务和服务访问点

实体：任何可以发送或接受的硬件或软件进程。
协议：控制两个对等层进行通信的规则集合。
服务：下层为上层通过接口提供的功能。
服务原点：上层使用下层提供的服务必须通过与下层交换一些命令,这些命令在OSI中成为“服务原点”。
服务访问点(SAP)：同一系统中相邻两层的实体进行交换的地方。
服务数据单元：层与层之间交换的数据单元。

协议有一个很重要的特点，比如把所有不利的情况都预先估计到，而不能假定一切都是正常的或者理想情况下的。
